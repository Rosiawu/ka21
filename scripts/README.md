# 脚本工具文档

本目录包含KA21工具导航项目的维护和管理脚本。

## 工具链接检查脚本 (check-urls.js)

此脚本用于检查工具列表中的所有URL是否能正常访问，帮助识别失效的工具链接。

### 功能特点

- 自动检查所有工具链接的可访问性
- 详细显示检查过程和结果
- 支持简洁模式，只显示最终摘要
- 生成失效链接报告，保存为JSON文件
- 提供进度指示和预计完成时间
- 支持自定义超时时间
- 彩色输出，提高可读性
- 智能错误识别与分类
- 预计完成时间动态更新
- 失效链接分类统计

### 使用方法

基本用法：

```bash
node scripts/check-urls.js
```

仅显示摘要结果：

```bash
node scripts/check-urls.js --check-only
```

设置自定义超时时间：

```bash
node scripts/check-urls.js --timeout=5000  # 设置超时为5秒
```

显示帮助信息：

```bash
node scripts/check-urls.js --help
# 或
node scripts/check-urls.js -h
```

### 选项说明

- `--help`, `-h`: 显示帮助信息
- `--check-only`: 只显示最终摘要，不显示检查过程 
- `--timeout=<毫秒>`: 设置请求超时时间，默认为10000毫秒(10秒)

### 输出解释

脚本运行后会输出以下信息：

1. 检查过程：显示每个URL的检查结果（正常访问或无法访问）
2. 摘要统计：显示正常工具和失效工具的数量
3. 失效工具列表：详细列出所有无法访问的工具及错误原因
4. 输出文件：生成`invalid-tools.json`文件，包含所有失效工具的详细信息

### 常见问题

1. **HTTP状态码403/429**: 这通常表示网站启用了防爬虫措施或IP被限制访问
2. **超时错误**: 可能是网站响应缓慢或者不可用
3. **ECONNREFUSED**: 表示服务器拒绝连接，可能是网站已关闭或变更
4. **证书错误**: HTTPS证书无效或过期

### 维护说明

如果脚本运行中发现大量误报（特别是403/429错误），可能需要添加以下改进：

1. 添加用户代理标头
2. 实现请求重试机制
3. 添加IP代理轮换
4. 增加更多请求头模拟真实浏览器

### 注意事项

- 此脚本仅用于检查链接可访问性，不会修改项目数据
- 某些网站可能有反爬虫措施，可能会出现误报
- 建议定期运行此脚本检查工具链接状态

### 优化建议

以下是进一步优化脚本的建议：

1. **增强请求模拟**
   - 添加随机用户代理列表，每次请求随机选择
   - 实现请求头完整模拟，包括referer、accept等常见头信息
   - 添加cookie支持，处理需要cookie的网站

2. **错误处理增强**
   - 按错误类型对失效链接进行分类统计
   - 区分暂时性错误和永久性错误
   - 对不同错误类型给出针对性修复建议

3. **批量处理功能**
   - 支持指定检查特定分类的工具
   - 实现批量重试失效链接
   - 添加对比模式，对比两次检查的结果差异

4. **报告增强**
   - 生成HTML格式报告，提高可读性
   - 添加图表可视化失效链接分布
   - 支持发送邮件通知功能
   - 增加历史记录功能，跟踪工具可用性变化趋势

5. **性能优化**
   - 实现并行请求处理，加快检查速度
   - 添加请求队列和速率限制，避免触发网站反爬虫机制
   - 实现智能超时控制，根据网站响应速度动态调整

6. **集成功能**
   - 与CI/CD流程集成，自动定期检查
   - 提供API接口，支持其他系统调用
   - 在失效工具上标记临时不可用状态
