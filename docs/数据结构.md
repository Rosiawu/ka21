# 数据结构

本文档详细说明了数字生命卡兹克-KA21工具导航平台使用的数据结构、类型定义和分类系统。

## 分类系统说明

### 有效的工具类型值（toolCategory）
按照功能类型对工具进行分类，每个工具必须且只能属于一个工具类型：

1. `writing` - AI写作工具
2. `image` - AI图像工具
3. `video` - AI视频工具
4. `office` - AI办公工具
5. `design` - AI设计工具
6. `chat` - AI对话聊天
7. `coding` - AI编程工具
8. `audio` - AI音频工具
9. `dev-platform` - AI开发平台
10. `agent` - AI通用智能体
11. `podcast` - AI播客工具
12. `misc` - 其他AI工具
13. `utils` - 四次元小工具（效率类小工具）

注意：虽然每个工具只能选择一个工具类型，但可以通过tags标签系统添加多个相关标签来展示工具的其他特性。

## 核心数据结构

### 工具（Tool）
```typescript
interface Tool {
  id: string;              // 工具唯一标识
  name: string;            // 工具名称
  description: string;     // 工具描述
  icon?: string;           // 工具图标URL（兼容旧字段）
  icons?: {                // 工具图标（新结构，优先）
    svg?: string;         // SVG格式图标（可选）
    png?: string;         // PNG格式图标（可选）
  };
  url: string;            // 工具官网URL
  tags: string[];         // AI功能标签
  toolCategory?: ToolCategoryId; // 工具类型分类（按功能类型分类，如"AI写作工具"、"AI视频工具"等）
  recommendLevel?: RecommendLevel; // 推荐等级（high/medium/low）
  accessibility?: AccessibilityType; // 可访问性（"直接访问"/"需要代理"）
  displayOrder?: number;  // 排序优先级（数字越小越靠前）
  guides?: Guide[];        // 使用指南
  relatedArticles?: RelatedArticle[]; // 相关文章
  relatedTutorials?: string[]; // 关联教程ID（新结构）
  groupComments?: GroupComment[]; // 群友点评
  createdAt?: string;      // 创建时间
  updatedAt?: string;      // 更新时间
  isVisible?: boolean;     // 是否在前端展示（支持精品轮动）
}
```

### 指南（Guide）


### 群友评价（GroupComment）
```typescript
interface GroupComment {
  content: string;     // 评价内容
  author: string;      // 评价作者
  createdAt: string;   // 评价时间
  reviewType?: string; // 评价类型（纯人工测评/无广告/为爱发电）
}
```

### 相关文章（RelatedArticle）
```typescript
interface RelatedArticle {
  title: string;       // 文章标题
  url: string;        // 文章链接

  publishDate?: string; // 发布日期（可选）
}
```

```typescript
type RecommendLevel = 'high' | 'medium' | 'low';
```

### 可访问性类型（AccessibilityType）
```typescript
type AccessibilityType = '直接访问' | '需要代理';
```

### 教程（Tutorial）
```typescript
interface Tutorial {
  id: string;         // 教程唯一标识
  title: string;      // 教程标题
  description: string; // 教程描述
  url: string;        // 教程链接
  author: string;     // 作者/来源
  publishDate: string; // 发布日期（YYYY-MM-DD）
  category: string;   // 教程分类
  difficultyLevel: '小白入门' | '萌新进阶' | '高端玩家'; // 难度级别
  skillTags: string[]; // 技能标签
  imageUrl: string;   // 教程图片URL（缺省时自动生成渐变图）
  recommendReason?: string; // 推荐理由
  relatedTools?: string[]; // 关联工具ID
}
```

## 数据关系

数据之间的主要关系包括：

1. **工具与分类**：每个工具通过 `toolCategory` 字段关联到一个工具分类
2. **工具与标签**：每个工具可以有多个标签，通过 `tags` 数组关联
3. **工具与指南**：每个工具可以有多个使用指南，通过 `guides` 数组关联
4. **工具与评价**：每个工具可以有多个群友评价，通过 `groupComments` 数组关联
5. **工具与教程/文章**：
   - 新：通过 `relatedTutorials`（教程ID列表）关联到 `tutorials.json` 中的教程
   - 旧：`relatedArticles` 仍受支持（兼容层）
6. **教程与分类**：每个教程通过 `category` 字段关联到一个教程分类

## 数据校验

为确保数据质量，项目实现了一系列数据验证机制：

1. **必填字段验证**：确保每个工具都有ID、名称、描述和URL等必要信息
2. **精选配置校验**：构建前自动验证精选配置与工具数据一致性（`scripts/validate-featured.ts`）
2. **类型检查**：确保数据字段符合预期的类型定义
3. **格式验证**：验证URL格式、日期格式等
4. **枚举值验证**：确保分类、推荐等级等字段使用预定义的有效值
5. **数据一致性检查**：确保关联数据的完整性和正确性

## 数据更新机制

项目提供了多种数据维护和更新的机制：

1. **手动编辑**：直接编辑 `src/data/tools.json` 和其他数据文件
2. **脚本工具**：使用 `scripts` 目录中的工具脚本进行批量数据处理
3. **自动同步**：从相关文章中自动提取教程数据等自动化流程
4. **验证脚本**：使用 `validate-tools.js` 脚本确保数据更新后的质量
