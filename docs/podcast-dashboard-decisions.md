# 播客播放量页面决策记录

## 2026-07-11：历史数据按需分页展示

- 背景：播放量页面曾在首屏一次生成所有单集、所有日期和所有平台的历史表格。按当时 33 集、99 个快照计算，会生成约 1.7 万个表格单元格，即使历史区域折叠也已存在于 DOM，导致页面打开和图表调整尺寸时明显晃动。
- 决策：单集表现每页展示 10 集；快照历史和所选单集历史每页展示 15 条；历史区域仅在用户展开后渲染；单集历史改为“节目选择器 + 趋势图 + 分页明细”。图表 resize 使用 160ms 防抖，仅在移动端断点变化时重新生成配置。
- 保持不变：`snapshots.json`、单集缓存、API 返回结构和历史快照兼容性均不变。
- 未采用：只用 CSS 隐藏完整历史 DOM，因为它不会减少初始节点创建；完全移除历史明细并仅提供下载，因为会降低日常查看数据的便利性。
- 回滚：回退 `public/podcast-dashboard/index.html`、`app.js` 和 `styles.css` 的对应提交即可，无需迁移或回填数据。

## 2026-07-14：首屏图表和 GPU 合成降载

- 背景：分页减少了表格 DOM，但页面仍会在首屏同步绘制全部 33 集 × 99 天的堆叠柱体，并在 Canvas、固定背景和多层毛玻璃之间进行 GPU 合成，Chrome 仍可能整页白屏。
- 决策：普通内容先渲染，图表延迟 180ms 初始化；日趋势只绘制最近 30 天，节目维度展示最新播放最高的 8 集，其余集数汇总为“其他”；去掉图表动画、柱体阴影和页面多层 `backdrop-filter`。
- 保持不变：原始快照、API 返回、分页历史和完整明细均保留，限制仅作用于首屏趋势图的可视化采样。
- 回滚：回退三个 dashboard 静态文件的对应改动即可，不涉及数据迁移。
