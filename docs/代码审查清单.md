# 代码审查检查清单

## 📋 使用说明

本检查清单用于确保所有代码提交都符合KA21工具导航项目的质量标准。请在代码审查时逐项检查。

## 🔍 基础检查

### 代码格式和规范
- [ ] 代码通过了ESLint检查，无错误和警告
- [ ] TypeScript编译无错误，类型定义正确
- [ ] 组件和函数命名符合项目规范（PascalCase/camelCase）
- [ ] 文件结构遵循项目目录规范
- [ ] 导入语句按照规范排序（React -> Next.js -> 项目内部 -> 第三方库）

### 代码质量
- [ ] 函数单一职责，逻辑清晰
- [ ] 变量命名语义明确，避免缩写和模糊命名
- [ ] 没有重复代码，遵循DRY原则
- [ ] 复杂逻辑有适当的注释说明
- [ ] 错误处理完善，边界情况考虑周全

## 🧩 组件开发检查

### 组件结构
- [ ] 组件文件结构符合规范（类型定义 -> 组件实现 -> 导出）
- [ ] Props接口定义完整，包含必需和可选属性
- [ ] 组件有适当的默认值设置
- [ ] 使用了正确的React Hooks顺序
- [ ] 避免了不必要的重渲染（适当使用useMemo/useCallback）

### 可复用性
- [ ] 组件设计具有良好的可复用性
- [ ] 通过props控制组件行为，避免硬编码
- [ ] 组件职责单一，不包含过多业务逻辑
- [ ] 支持必要的自定义样式和行为扩展

### 性能优化
- [ ] 列表渲染使用了稳定且唯一的key值
- [ ] 图片使用Next.js Image组件进行优化
- [ ] 大型组件适当拆分，避免单个组件过于复杂
- [ ] 状态更新逻辑高效，避免不必要的state变更

## 🎨 UI和样式检查

### 响应式设计
- [ ] 在所有断点下显示正常（sm/md/lg/xl/2xl）
- [ ] 移动端交互友好，触摸目标足够大
- [ ] 文字在小屏幕下仍然可读
- [ ] 布局在极端内容长度下不会破坏

### 样式实现
- [ ] 使用Tailwind CSS类名，避免自定义CSS（除非必要）
- [ ] 颜色使用CSS变量系统，支持主题切换
- [ ] 动画和过渡效果自然，不影响性能
- [ ] 遵循设计系统的spacing和sizing规范

### 暗色模式
- [ ] 所有UI元素都适配暗色主题
- [ ] 文字对比度符合可访问性标准
- [ ] 图片和图标在暗色模式下显示正常
- [ ] 品牌色彩在两种主题下都保持一致性

## 📊 数据处理检查

### 类型安全
- [ ] 所有数据接口都有完整的TypeScript类型定义
- [ ] API响应数据进行了适当的类型验证
- [ ] 避免使用any类型，使用具体类型或泛型
- [ ] 数据转换函数有适当的类型守卫

### 数据验证
- [ ] 外部数据源（JSON文件、API）进行了验证
- [ ] 用户输入进行了清洗和验证
- [ ] 边界条件处理得当（空数组、null值等）
- [ ] 数据格式转换逻辑正确且健壮

### 状态管理
- [ ] 状态提升到合适的组件层级
- [ ] 避免了不必要的全局状态
- [ ] 状态更新逻辑清晰，易于追踪
- [ ] 异步操作有适当的加载和错误状态

## ♿ 可访问性检查

### 语义化HTML
- [ ] 使用了语义化的HTML标签
- [ ] 表单元素有正确的label关联
- [ ] 列表项使用了正确的列表标签
- [ ] 标题层级结构合理（h1-h6）

### ARIA支持

- [ ] 动态内容变化有ARIA live regions
- [ ] 复杂组件有适当的role属性
- [ ] 屏幕阅读器可以正确理解页面结构

### 键盘导航
- [ ] 所有交互元素可以通过键盘访问

- [ ] 焦点状态清晰可见
