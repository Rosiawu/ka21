### 🧩 组件开发模板
```typescript
'use client'; // 仅客户端组件需要

import { useState } from 'react';
import { ComponentProps } from '@/lib/types';

interface MyComponentProps {
  title: string;
  count?: number;
  onAction?: () => void;
}

/**
 * 组件功能描述
 * @param title - 标题文本
 * @param count - 数量，默认为0
 * @param onAction - 点击回调
 */
export default function MyComponent({ 
  title, 
  count = 0, 
  onAction 
}: MyComponentProps) {
  const [isActive, setIsActive] = useState(false);
  
  const handleClick = () => {
    setIsActive(!isActive);
    onAction?.();
  };
  
  return (
    <div className="bg-white dark:bg-slate-800 rounded-lg p-4">
      <h2 className="text-lg font-semibold">{title}</h2>
      <p className="text-sm text-neutral-600 dark:text-neutral-400">
        Count: {count}
      </p>
      <button 
        onClick={handleClick}
        className="mt-2 px-4 py-2 bg-primary-500 text-white rounded hover:bg-primary-600 transition-colors"
      >
        {isActive ? '激活' : '未激活'}
      </button>
    </div>
  );
}
```

### 📊 类型定义规范
```typescript
// ✅ 基础类型
export type Tag = string;
export type RecommendLevel = "high" | "medium" | "low";

// ✅ 接口定义
export interface Tool {
  id: string;                    // 必需字段
  name: string;
  description: string;
  url: string;
  tags: Tag[];
  recommendLevel?: RecommendLevel; // 可选字段
  isVisible?: boolean;            // 默认true
}

// ✅ 类型守卫
export function isValidTool(data: unknown): data is Tool {
  return typeof data === 'object' && 
         data !== null && 
         typeof (data as Tool).id === 'string';
}
```

## 🔧 常用开发任务

### 添加新组件
1. 在`src/components/`下创建组件文件
2. 定义Props接口
3. 添加组件文档注释
4. 实现组件逻辑
5. 导出组件
6. 在需要的地方导入使用

### 添加新页面
1. 在`src/app/`下创建目录
2. 添加`page.tsx`文件
3. 可选：添加`loading.tsx`和`error.tsx`
4. 配置页面元数据
5. 更新导航链接

### 修改数据结构
1. 更新`src/lib/types.ts`中的类型定义
2. 修改相关验证函数`src/lib/validate.ts`
3. 更新数据文件格式
4. 修改使用该数据的组件
5. 运行测试确保无破坏性变更

### 添加新的工具分类
1. 更新`src/lib/types.ts`中的`ToolCategoryId`
2. 在`src/data/toolCategories.ts`中添加分类配置
3. 更新相关组件的图标映射
4. 添加分类图标文件
5. 测试分类筛选功能

## 🧪 测试和调试

### 本地测试流程
```bash
# 代码格式检查
npm run lint

# 类型检查
npx tsc --noEmit

# 启动开发服务器
npm run dev

# 构建测试
npm run build
```

### 常见问题排查

1. **TypeScript编译错误**
   - 检查类型定义是否正确
   - 确认导入路径无误
   - 查看是否缺少必需的props

2. **样式不生效**
   - 确认Tailwind类名拼写正确
   - 检查是否需要暗色模式适配
   - 验证CSS变量是否定义

3. **组件不渲染**
   - 检查客户端/服务器组件声明
   - 确认数据格式正确
   - 查看浏览器控制台错误
