# 开发指南

本文档提供了数字生命卡兹克-KA21工具导航平台的详细开发流程、编码规范和贡献指南。

## 开发环境设置

### 前提条件
- Node.js 18.x 或更高版本
- npm 9.x 或更高版本
- Git

### 初始设置
1. 克隆项目仓库
```bash
git clone [仓库URL]
cd ka21-tools
```

2. 安装依赖
```bash
npm install
```

3. 启动开发服务器
```bash
npm run dev
```

4. 在浏览器中访问 [http://localhost:3000](http://localhost:3000)

## 开发流程

### 分支管理
- **main**: 生产环境分支，保持稳定
- **develop**: 开发分支，所有功能开发完成后合并到此分支
- **feature/\***: 用于开发新功能的分支
- **fix/\***: 用于修复bug的分支

### 开发新功能
1. 从 develop 分支创建新的 feature 分支
```bash
git checkout develop
git pull
git checkout -b feature/your-feature-name
```

2. 开发功能并测试
3. 确保代码通过 ESLint 检查
```bash
npm run lint
```

4. 提交更改
```bash
git add .
git commit -m "feat: 添加新功能描述"
```

5. 推送到远程仓库
```bash
git push origin feature/your-feature-name
```

6. 创建 Pull Request 到 develop 分支

## 编码规范

### TypeScript 和 React
- 使用 TypeScript 编写所有组件和功能
- 遵循 React 函数组件和 Hooks 最佳实践
- 使用 Next.js 的 App Router 架构

### 样式和 UI
- 使用 Tailwind CSS 进行样式设计
- 确保响应式设计，支持移动设备
- 保持与现有 UI 设计语言一致

### 代码风格
- 遵循项目 ESLint 规则配置
- 使用有意义的变量和函数名
- 代码添加适当的注释

## 贡献指南

欢迎提交Pull Request来改进项目：

1. Fork 项目到个人仓库
2. 创建功能分支
3. 提交更改
4. 推送到分支
5. 创建Pull Request

### Pull Request 规范
- 清晰描述更改的目的和实现方法
- 确保代码通过 ESLint 检查
- 如适用，添加必要的单元测试
- 确保文档保持更新

## 项目结构指南

在编写新组件或修改现有代码时，请遵循项目的结构组织方式：

### 组件
- 所有组件放在 `src/components/` 目录下
- 组件文件使用 PascalCase 命名（如 `ToolCard.tsx`）
- 相关的小型组件可以放在同一文件中

### 页面
- 页面组件放在 `src/app/` 目录下，对应路由结构
- 使用 Next.js 14 App Router 的约定（如 `page.tsx`、`layout.tsx`）

### 工具函数
- 通用工具函数放在 `src/lib/` 目录下
- 按功能分组，保持文件的专注性

### 数据文件
- 数据文件存放在 `src/data/` 目录下
- 遵循现有的数据结构和格式

## 非前端开发者指南

如果你不熟悉前端开发，以下是一些使用和维护本项目的建议:

1. **添加或更新工具**:
   - 直接编辑 `src/data/tools.json` 文件
   - 遵循现有的JSON格式添加新工具
   - 确保包含必要字段: id, name, description, url, tags, toolCategory
   - 可选字段包括: icon, recommendLevel, accessibility, guides, relatedArticles, groupComments

2. **更新工具分类**:
   - 编辑 `src/data/toolCategories.ts` 文件
   - 遵循现有格式添加新分类
   - 历史脚本 `scripts/update-tool-categories.js` 已归档，如需批量处理请参考现有分类映射手动更新

3. **添加教程内容**:
   - 更新 `src/data/tutorials.ts` 文件
   - 确保图片放在正确的公共目录

4. **修复数据问题**:
   - 使用 `scripts/validate-tools.js` 脚本验证数据完整性
   - 使用 `scripts/fix-tools.js` 脚本修复常见数据问题

5. **样式调整**:
   - 主要样式在 `src/app/globals.css` 中定义
   - 组件特定样式在各自的组件文件中使用 Tailwind 类名实现

6. **理解项目架构**:
   - Next.js 14 使用 App Router 架构
   - `src/app` 目录中的文件直接对应网站路由
   - 组件分为服务器组件和客户端组件
   - 数据主要是静态JSON，无需数据库

## 构建和部署

1. 构建生产版本
```bash
npm run build
```

2. 本地预览生产版本
```bash
npm run start
```

3. 使用 Vercel 部署
项目配置为使用 Vercel 进行自动部署。每次推送到 main 分支时，Vercel 将自动构建和部署最新版本。
