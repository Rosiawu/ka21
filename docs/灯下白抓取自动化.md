# 灯下白抓取自动化

这套抓取已经收进当前仓库，不再依赖你本机 `Downloads/...` 里的私有路径。

## 现在能做到什么

- 每天自动抓一次灯下白公开播放数据
- 手机上手动触发一次抓取
- 抓完后自动提交到 GitHub
- 如果仓库接了 Vercel，新的数据 commit 会顺带触发部署

## 数据文件

- `data/podcast-dashboard/config.json`
- `data/podcast-dashboard/episodes-cache.json`
- `data/podcast-dashboard/snapshots.json`

## 本地手动抓取

```bash
npm run podcast:capture-snapshot
```

只看结果、不写文件：

```bash
npm run podcast:capture-snapshot -- --dry-run
```

`--dry-run` 只输出抓取结果，不会写入 `episodes-cache.json` 或 `snapshots.json`。

## 手机上怎么手动触发

1. 手机上打开 GitHub 仓库 `Rosiawu/ka21`
2. 进入 `Actions`
3. 打开 `Update Podcast Dashboard Snapshot`
4. 点 `Run workflow`

这次 workflow 会直接跑抓取脚本，并把 `snapshots.json` 的更新自动 commit + push 回仓库。

## 自动时间

- GitHub Actions 已配置为每天北京时间 `10:10` 自动执行一次（GitHub cron 使用 UTC，对应 `02:10 UTC`）
- 每次自动抓取都会先从 RSS 刷新节目列表；第 28 期及之后的新节目会自动写入 `episodes-cache.json`，并随快照一起 commit 回仓库。
- Actions 日志会打印当前节目总数和最新一期标题，方便确认是否已经提取到最新节目。

## 当前公开口径

当前脚本能抓到公开播放量的平台：

- 小宇宙
- 喜马拉雅
- 网易云音乐
- 荔枝
- 蜻蜓 FM（当前能抓频道总播放，单集公开可见度不完整）

当前公开页拿不到播放量的平台：

- Apple 播客
- Spotify
- YouTube（当前公开频道页没有匹配到《灯下白》节目视频播放量时，不再写入 `0`）
