## 二、标准流程（新手照抄版）

### 第 1 步：先确认最近改了什么

```bash
git log --since="2026-02-25" --oneline
```

作用：先看清最近提交，避免“改着改着不知道自己在哪个版本”。

### 第 2 步：本地预览，确认效果

```bash
npm run dev
```

打开页面后重点检查这 3 项：

1. 输入框视觉：像对话框，不像搜索框。
2. 英文语言下：页面文案是否仍有中文。
3. 工具卡片：名称、描述、标签是否都已英文化。

### 第 3 步：确认无误再提交

```bash
git add .
git commit -m "feat: your message"
```

核心原则：先预览，再 commit。这样返工最少。

### 第 4 步：推送到 GitHub

先把远程改为 SSH：

```bash
git remote set-url origin git@github.com:Rosiawu/ka21.git
```

如果网络不稳定，先切 HTTP/1.1 再推送：

```bash
git config http.version HTTP/1.1
git push origin main
```

## 三、推送失败时怎么排查（超实用）

### A. 先测网络连通

```bash
curl -I -4 https://github.com
curl -I -6 https://github.com
```

判断方法：

1. `-4` 通、`-6` 不通：通常是 IPv6 路径问题，可先走 IPv4。
2. 两个都不通：先换网络（你这次切手机热点就是正确动作）。

### B. 测 SSH 是否可用

```bash
ssh -T git@github.com
```

看到欢迎信息（successfully authenticated）就表示 SSH 通路 OK。

### C. 没有 key 就生成 key

```bash
ssh-keygen -t ed25519 -C "你的邮箱"
cat ~/.ssh/id_ed25519.pub
```

把公钥复制到 GitHub -> Settings -> SSH keys。

你这次截图里显示：

1. key 已存在
2. 权限是 Read/write
3. Added on Feb 25, 2026

说明这一步是完成的。
