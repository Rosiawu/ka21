# 热门推荐板块功能文档（方案 A）

版本：v1.0  
日期：2025-09-08  
状态：✅ 开发完成，已上线

## 1. 背景与目标
- 在首页“搜索框下方、教程轮播上方”新增“热门推荐”板块，提升工具曝光与转化。
- 采用配置驱动与静态数据，首屏 SSR 稳定，交互侧在客户端实现“换一换”。

## 2. 范围与约束
- 技术栈：Next.js 14 + TypeScript + Tailwind CSS。
- 数据来源：仅静态 JSON（无后端数据库/实时 API）。
- 响应式：移动端显示 3 个，桌面端显示 4 个；不足不填空位，禁用"换一换"。
- 会话去重：会话内同一分类尽量不重复，跨分类允许重复。

## 3. 位置与布局
- 位置：位于首页“搜索框下方，教程轮播上方”。
- 结构：
  - 区块标题“热门推荐”与描述文案（可选）。
  - 三个分类卡片（例如：AI 创作助手、效率办公工具、学习成长工具）。
  - 每个分类包含：标题、副标题、图标、工具网格、按钮（换一换 / 查看全部）。

## 4. 数据契约（data/featured.json）
顶层结构：
```json
{
  "version": "1.0",
  "order": ["ai_creation", "productivity", "learning"],
  "categories": {
    "ai_creation": {
      "title": "AI创作助手",
      "subtitle": "艺术新风潮，创意无边界",
      "icon": "🤖",
      "featured_tools": ["midjourney", "jimeng", "qiyu", "stable-diffusion"],
      "backup_tags": ["图像生成", "AI艺术", "创意设计"],
      "backup_categories": ["image-generation", "creative"],
      "view_all_href": "/tools?tags=AI艺术,图像生成",
      "order": 10
    }
    // 其余分类...
  }
}
```

字段说明：
- `version`：配置版本号，用于标识变更。
- `order`：分类展示顺序（为 `categories` 的 key 列表）。
- `categories`：各分类配置映射。
  - `title`：分类标题（中文）。
  - `subtitle`：分类副标题（简短卖点）。
  - `icon`：分类图标（emoji 或站内 icon key）。
  - `featured_tools`：工具 ID 列表（必须与 `data/tools.json` 的 `id` 一致且分类内唯一）。
  - `backup_tags`（可选）：兜底标签，基于现有工具标签体系。
  - `backup_categories`（可选）：兜底分类，映射至 `toolCategories.ts` 中已存在的分类键。
  - `view_all_href`：查看全部链接（站内路径 + 查询参数）【已移除，保留用于配置兼容性】。
  - `order`（可选）：数值越小越靠前，用于精细排序（如指定则优先生效）。

不变式（构建期校验约束）：
- `featured_tools` 中的每个 ID 必须存在于 `data/tools.json`。
- 同一分类内 `featured_tools` 不可重复；`order` 中的 key 必须出现在 `categories`。
- `view_all_href` 必须为站内路径，以 `/` 开头。

## 5. 选择与兜底逻辑
首屏 SSR（稳定、可索引）：
- 先严格按 `featured_tools` 的原始顺序选择。
- 若不足当前页容量（移动 3、桌面 4），则从兜底池补齐：
  - 兜底池=满足 `backup_tags` 或 `backup_categories` 的工具集合。
  - 兜底排序：`recommendLevel` 降序（high > medium > low），再按名称升序。
  - 分类内去重：排除已选 `featured_tools`。

客户端“换一换”（每分类独立）：
- 构建“牌堆”：`featured_tools` 在前，兜底池在后（同过滤与去重）。
- 若 `total <= pageSize`，隐藏/禁用“换一换”。
- 会话内洗牌：使用 Fisher–Yates 算法对牌堆进行一次随机打乱，并分页滚动；耗尽后重洗重置。

## 6. 状态与持久化（会话范围）
- 页容量 `pageSize`：
  - 客户端用 `matchMedia('(min-width: 768px)')` 判断：桌面 4，否则 3。
  - SSR 首屏固定按当前断点策略输出，不进行随机化。
- `sessionStorage` 键约定：
  - `hotrec:<category>:deck`：洗牌后的牌堆（序列化 ID 数组）。
  - `hotrec:<category>:pageIndex`：当前页下标（从 0 开始）。
  - `hotrec:<category>:pageSize`：记录上次使用的页容量；若变化需重建分页。

## 7. 交互与可用性
- 悬停：保留现有 hover，但降低动效强度；遵循 `prefers-reduced-motion`。
- 按钮：
  - “换一换”按分类独立；`aria-label` 包含分类标题。
  - “查看全部”直达 `view_all_href`。
- 卡片：使用 `<a>` 元素，名称为工具标题；键盘可达（Tab 导航、Enter/Space 生效）。

## 8. 埋点规范（lib/analytics.ts 包装）
事件与载荷：
- `hot_shuffle`
  - 字段：`{ category_key, category_title, page_index_after, total_tools, page_size }`
  - 触发：用户点击某分类的“换一换”后。
- `hot_click`
  - 字段：`{ category_key, category_title, tool_id, position, from: "hot_section" }`
  - 触发：用户点击热门推荐中的工具卡片。

原则：
- 仅在客户端上报；同一用户动作仅上报一次；`position` 为 1 基（更贴近可视顺序）。
- 兼容现有统计（如 Umami/Clarity）；统一通过 `lib/analytics.ts` 转发。

## 9. 构建期校验（scripts/validate-featured.ts）
目标：在 `next build` 前阻断明显配置错误，降低线上故障。
- Schema 校验（建议 Zod）：`featured.json` 结构与类型合法。
- ID 交叉校验：`featured_tools` 必须存在于 `data/tools.json`。
- 一致性：`order` 与 `categories` 对应；分类内不重复。
- 链接合法性：`view_all_href` 站内路径校验。
- 兜底充足度：若兜底池过小（< 页容量），给出警告但不阻断构建。

## 10. 组件与文件（已实现）
- 数据：`data/featured.json`（配置文件，包含三个分类的工具配置）。
- 类型：`types/featured.ts`（`FeaturedConfig`, `FeaturedCategory`）。
- 工具：`lib/featured.ts`（选择/兜底/洗牌/分页）、`lib/hot-analytics.ts`（埋点）。
- 组件：
  - `components/hot/HotSection.tsx`（板块容器；服务端组件）
  - `components/hot/HotCategory.tsx`（分类卡片；客户端交互）
  - `components/hot/HotToolCard.tsx`（工具项）
- 集成：`src/components/HomeContent.tsx` 在搜索框下方、教程轮播上方插入 `HotSection`。

## 11. 渲染与性能
- 首屏：SSR 严格按配置与兜底排序，避免水合抖动。
- 图片与图标：懒加载；`icon` 无效时显示统一占位。
- 依赖：不新增重型库；静态导入 JSON。

## 12. QA 验收清单
- SSR 首屏稳定，无水合警告。
- 桌面显示 6，移动显示 4；窗口变化时分页行为正确。
- 当可展示工具数 ≤ 页容量时，“换一换”隐藏或禁用。
- 每分类在一次会话内尽量不重复，耗尽后重洗。
- `hot_shuffle` 与 `hot_click` 事件载荷正确、无重复上报。
- “查看全部”跳转路径正确，包含预期 query。
- 图标缺失或不合法时有明确回退；hover 动效轻量、可达性良好。
