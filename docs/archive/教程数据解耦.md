# 教程数据解耦方案文档

本文档详细介绍了教程数据解耦的设计、实现和使用方法。

## 背景

在原设计中，教程数据(相关文章)作为工具数据的一部分，嵌套在`tools.json`文件的`relatedArticles`字段中。这种设计存在以下几个问题：

1. **数据耦合度高**：教程数据与工具数据紧密绑定，不易独立管理
2. **重复问题**：如果一篇教程涉及多个工具，需要在每个工具中重复定义
3. **扩展性受限**：难以为教程添加更丰富的元数据和关系信息

## 解耦方案

新方案将教程数据从工具数据中分离出来，独立存储和管理：

1. **数据分离**：创建独立的`tutorials.json`文件存储所有教程数据
2. **双向关联**：通过ID建立教程与工具的双向关联关系
3. **平稳过渡**：通过配置开关支持新旧两种数据源，确保平稳迁移

## 数据结构

### 教程数据结构 (tutorials.json)

```json
{
  "tutorials": [
    {
      "id": "tutorial-mp-weixin-qq-com-title-summary",
      "title": "教程标题",
      "url": "https://mp.weixin.qq.com/s/...",
      "source": "作者/来源",
      "publishDate": "2025-02-24", 
      "difficultyLevel": "小白入门", // 可选值: "小白入门" | "萌新进阶" | "高端玩家"
      "category": "AI大模型", // 教程分类
      "skillTags": ["ChatGPT", "提示词工程"], // 技能标签
      "recommendReason": "推荐理由说明",
      "customImageUrl": "/images/tutorials/custom.jpg", // 可选, 自定义图片
      "relatedTools": ["tool-id-1", "tool-id-2"] // 关联工具ID列表
    }
  ],
  "meta": {
    "version": "1.0.0",
    "updatedAt": "2025-02-24T12:00:00Z",
    "count": 42
  }
}
```

### 工具数据更新 (tools-updated.json)

在新方案中，工具数据中的`relatedArticles`字段被替换为`relatedTutorials`字段，存储相关教程的ID列表：

```json
{
  "id": "deepseek",
  "name": "DeepSeek",
  // ... 其他工具字段
  "relatedTutorials": [
    "tutorial-mp-weixin-qq-com-title-1",
    "tutorial-mp-weixin-qq-com-title-2"
  ],
}
```

## 数据源切换与使用

系统支持新旧两种数据源，通过配置文件`src/lib/config.ts`中的`USE_NEW_TUTORIALS_SOURCE`来控制：

```typescript
// 在src/lib/config.ts中设置
export const USE_NEW_TUTORIALS_SOURCE = true; // true 使用 tutorials.json（推荐）
```
当启用新数据源时，`src/data/tutorials.ts` 会：
- 从 `tutorials.json` 加载并映射为前端使用的 `Tutorial` 结构（将 `source` 标准化为 `author`，处理 `customImageUrl` 等）。
- 当缺少图片时，自动生成渐变 SVG 占位图（按分类与顺序生成）。
- 提供 `findTutorialById` 与（兼容旧逻辑的）`findTutorialBySlug`。

## 使用建议

1. **迁移建议**：
   - 先运行迁移脚本生成独立的教程数据
   - 验证数据完整性和正确性
   - 测试两种数据源下的应用功能
   - 确认无误后，可以启用新数据源

2. **教程数据维护**：
   - 直接编辑 `src/data/tutorials.json` 添加或更新教程。
   - 设置唯一 `id`（推荐：`tutorial-域-标题关键信息`）；完善 `author`/`publishDate`/`category`/`difficultyLevel`/`skillTags` 等字段。
   - 通过 `relatedTools` 关联涉及的工具（使用工具 `id`）。

3. **工具关联教程**：
   - 在工具数据中使用 `relatedTutorials` 字段列出相关教程 ID；页面优先根据 ID 在 `tutorials.json` 中查找详情。
   - 对于未迁移的工具，仍支持读取旧的 `relatedArticles`（兼容层）。

## 故障排除

如果在使用新数据源时遇到问题：

1. 检查 `src/data/tutorials.json` 是否存在且结构正确。
2. 确认 `src/lib/config.ts` 中 `USE_NEW_TUTORIALS_SOURCE` 为 `true`。
3. 查看浏览器控制台与服务器日志中的报错提示。
4. 暂时可切回旧数据源（将配置改为 `false`）进行对比排查。
