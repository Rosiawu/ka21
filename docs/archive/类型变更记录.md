# KA21工具导航 - 类型变更记录

本文档记录了项目中的类型变更历史，帮助开发团队了解数据结构的演进过程及其影响。

## 业务功能分类添加 (2024-02-26)

### 变更内容
- 添加了`BusinessCategoryId`类型和`BusinessCategory`接口，支持Futurepedia风格的业务分类
- 在`Tool`接口中添加了`businessCategory`字段，用于标识工具所属的业务分类
- 创建了业务分类数据文件和业务分类筛选组件

### 前后对比
```typescript
// 新增的类型
export type BusinessCategoryId = 
  | "customer-support" 
  | "sales" 
  | "back-office" 
  | "operations" 
  | "growth-marketing" 
  | "writing-editing" 
  | "technology-it" 
  | "design-creative" 
  | "workflow-automation";

export interface BusinessCategory {
  id: BusinessCategoryId;
  name: string;
  icon: string;
}

// 变更后的Tool接口
interface Tool {
  // ...其他字段
  businessCategory?: BusinessCategoryId; // 新增字段
  // ...其他字段
}
```

### 迁移方案
- 使用脚本根据工具标签自动为工具添加对应的业务分类
- 对于没有匹配标签的工具，默认分配到"技术与IT"分类
- 后续可以根据实际需求手动调整工具的业务分类

## 图标结构优化 (2024-02-25)

### 变更内容
- 在`Tool`接口中添加了`icons`对象字段，支持同时存储SVG和PNG格式的图标
- 保留了原有的`icon`字段以保持向后兼容性

### 前后对比
```typescript
// 变更前
interface Tool {
  // ...其他字段
  icon?: string;           // 工具图标URL
  // ...其他字段
}

// 变更后
interface Tool {
  // ...其他字段
  icon?: string;           // 工具图标URL（保留向后兼容性）
  icons?: {                // 新增的工具图标结构
    svg?: string;         // SVG格式图标URL
    png?: string;         // PNG格式图标URL
  };
  // ...其他字段
}
```

### 实现状态
- 类型定义已更新（`src/lib/types.ts`）
- 验证函数已更新（`src/lib/validate.ts`）
- 组件已支持新结构（`ToolCard.tsx`, `ToolDetail.tsx`, `page.tsx`）
- 当前代码会优先使用`icons.svg`，然后是`icons.png`，最后回退到`icon`

### 临时处理
由于当前数据中还没有`icons`字段，我们在组件中暂时直接使用`icon`字段：
```typescript
// 原本计划的逻辑
const iconUrl = icons?.svg || icons?.png || icon;

// 当前临时使用的逻辑
const iconUrl = icon;
```

### 迁移计划
1. **阶段1**：在新工具中开始使用`icons`对象（当前准备阶段）
2. **阶段2**：逐步将现有工具数据从`icon`迁移到`icons`结构
   - 可能需要编写脚本，根据文件扩展名自动判断是SVG还是PNG
3. **阶段3**：完成所有工具的迁移
4. **阶段4**：考虑在将来某个版本中废弃`icon`字段

### 影响范围
- `ToolCard.tsx` - 工具卡片组件的图标显示
- `ToolDetail.tsx` - 工具详情页的图标显示
- `page.tsx` - OpenGraph图像元数据

## 数据验证增强 (2024-02-25)

### 变更内容
- 增强了`validate.ts`中的数据验证逻辑
- 添加了对`icons`字段的验证
- 添加了对`groupComments`字段的完整验证
- 增强了对`relatedArticles`字段的验证

### 实现细节
```typescript
// icons字段验证
if (tool.icons !== undefined) {
  if (typeof tool.icons !== 'object') return false;
  if (tool.icons.svg !== undefined && typeof tool.icons.svg !== 'string') return false;
  if (tool.icons.png !== undefined && typeof tool.icons.png !== 'string') return false;
}

// groupComments字段验证
if (tool.groupComments !== undefined) {
  if (!Array.isArray(tool.groupComments)) return false;
  for (const comment of tool.groupComments) {
    if (!comment || typeof comment !== 'object') return false;
    if (!comment.content || typeof comment.content !== 'string') return false;
    if (comment.author !== undefined && typeof comment.author !== 'string') return false;
    if (comment.createdAt !== undefined && typeof comment.createdAt !== 'string') return false;
    if (comment.reviewType !== undefined && typeof comment.reviewType !== 'string') return false;
  }
}
```

### 影响范围
- 数据验证环节（如数据导入、API交互）
- 影响所有使用`validateTool`和`validateTools`函数的地方

## 类型变更指南

当需要对类型进行变更时，请遵循以下步骤：

1. 在`types.ts`中更新类型定义
2. 在`validate.ts`中添加相应的验证逻辑
3. 更新受影响的组件和函数
4. 更新本文档，记录变更内容和影响
5. 考虑是否需要数据迁移或转换
6. 更新测试用例以覆盖新的类型情况
