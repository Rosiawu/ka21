## KA21 多语言（i18n）实施计划（Locale-based routing | next-intl + App Router）

- **目标**: 采用基于路径前缀的多语言（如 `/zh/...`、`/en/...`），与 Next.js App Router 深度结合，获得清晰的 SEO 语义与可分享的多语言 URL。
- **方案**: 使用 next-intl 的 App Router 集成 + 顶级 `[locale]` 段路由；根路径 `/` 自动根据偏好/默认语言重定向到 `/{locale}`。
- **参考**: next-intl 官方 App Router 指南（Locale-based routing）`https://next-intl.dev/docs/getting-started/app-router`

---

### 一、范围与不改动项
- 不改动：工具与教程数据结构、热门推荐、搜索逻辑、Clarity/Umami 接入方式（已移除百度统计）。
- 主要变更：
  - 路由结构调整为 `app/[locale]/...`，所有页面进入该段下。
  - 根 `/` 做语言选择与重定向（默认 zh，可结合 Accept-Language 或 Cookie）。
  - UI 文案国际化（先覆盖公共与核心页面）。

### 四、next-intl 请求级配置（RSC）

- `src/i18n/request.ts`
```ts
import {getRequestConfig} from 'next-intl/server';

export default getRequestConfig(async ({locale}) => {
  // 容错：仅允许特定语言
  const supported = ['zh', 'en'] as const;
  const finalLocale = supported.includes(locale as any) ? locale : 'zh';

  return {
    locale: finalLocale,
    messages: (await import(`../../messages/${finalLocale}.json`)).default
  };
});
```

### 九、语言切换（基于路径前缀）

- 简单方案：在切换时替换路径前缀
```ts
// utils/i18n-path.ts
export function switchLocalePath(pathname: string, target: 'zh'|'en') {
  // 假设路径形如 "/{locale}/rest/of/path"
  const parts = pathname.split('/');
  parts[1] = target; // 覆盖 locale 段
  return parts.join('/') || '/';
}
```

```tsx
// 头部按钮（客户端）
"use client";
import {usePathname, useRouter} from 'next/navigation';
import {switchLocalePath} from '@/utils/i18n-path';

export function LocaleSwitcher() {
  const pathname = usePathname();
  const router = useRouter();

  return (
    <div className="flex gap-2">
      <button onClick={() => router.push(switchLocalePath(pathname, 'zh'))}>中文</button>
      <button onClick={() => router.push(switchLocalePath(pathname, 'en'))}>English</button>
    </div>
  );
}
```

- 可选：同时写入 cookie 便于首跳 `/` 识别。

### 十、Middleware（可选增强）

- 基于 `Accept-Language` / cookie 自动选择语言并首跳重定向
```ts
// middleware.ts（可选）
import {NextResponse} from 'next/server';
import type {NextRequest} from 'next/server';

const PUBLIC_FILE = /\.(.*)$/;

export function middleware(req: NextRequest) {
  const {pathname} = req.nextUrl;

  // 忽略静态、API 与已带 locale 的路径
  if (
    PUBLIC_FILE.test(pathname) ||
    pathname.startsWith('/api') ||
    /^\/(zh|en)(\/|$)/.test(pathname)
  ) {
    return NextResponse.next();
  }

  // 从 cookie 或头部推断
  const cookieLocale = req.cookies.get('locale')?.value;
  const acceptLang = req.headers.get('accept-language') || '';
  const preferred = cookieLocale || (acceptLang.startsWith('zh') ? 'zh' : 'en');

  const url = req.nextUrl.clone();
  url.pathname = `/${preferred}${pathname}`;
  return NextResponse.redirect(url);
}

export const config = {
  matcher: ['/((?!_next).*)']
};
```

### 十一、埋点/SEO/资源适配

- 埋点：在初始化处增加 `locale` 维度（例如 `setTag('locale', locale)`），不影响现有事件上报。
- SEO：
  - 每个页面可根据 `params.locale` 设置 `alternates.languages` 生成 `hreflang`。
  - 站点地图可扩展为多语言 URL。
- 资源：图片与静态资源路径不变；UI 文案改为按命名空间从 messages 获取。

### 十三、验收标准（DoD）
- `/zh/...` 与 `/en/...` 路由可正常访问，根 `/` 自动重定向到默认语言或识别语言。
- 核心页面 UI 文案可在两种语言间正确切换。
- 构建与类型检查通过，RSC/CSR 均正常。
- 埋点追加 `locale` 维度，SEO 可选添加 `hreflang`。
- 无静态资源 404、无水合告警（图片 `sizes` 等保持原有优化）。

---

### 十四、风险与回滚
- 风险：遗漏页面迁移导致 404；路径切换影响导航；命名空间键冲突。
- 规避：一次性完成路由迁移；建立 i18n key 规范（命名空间/键名）；PR 逐页 Review。
- 回滚：保留旧分支；如需回滚，恢复到无 `[locale]` 的目录结构并移除 Middleware。
