# 文档结构说明

本文档说明了数字生命卡兹克-KA21工具导航平台的文档结构组织方式。

## 文档组织结构

项目文档采用模块化结构，按主题分类整理，便于查找和维护：

1. **README.md** - 项目概览、快速开始、文档导航
2. **docs/功能列表.md** - 特色功能和已实现功能
3. **docs/功能列表-更多.md** - 更多已实现功能
4. **docs/技术架构.md** - 技术栈和项目结构
5. **docs/数据结构.md** - 数据结构和分类系统
6. **docs/开发指南.md** - 开发指南和贡献流程
7. **docs/脚本工具.md** - 脚本工具使用说明
8. **docs/部署指南.md** - 部署流程和最佳实践
9. **docs/优化建议.md** - 优化建议和最近优化总结
10. **docs/更新日志.md** - 项目版本和更新历史
11. **docs/文档结构.md** - 本文件，文档结构说明
12. **docs/I18n-多语言计划.md** - 多语言计划
13. **docs/项目状态.md** - 项目状态报告
14. **docs/快速入门指南.md** - 快速入门
15. **docs/代码审查清单.md** - 代码审查清单
16. **docs/自定义属性.md** - 自定义属性
17. **docs/教程数据解耦.md** - 教程数据解耦
18. **docs/热门推荐规范.md** - 热门推荐规范
19. **docs/Clarity-设置指南.md** - Clarity 配置
20. **docs/archive/工作日志.md** - 详细的项目工作日志（已归档）

## 文档更新指南

更新文档时请遵循以下原则：
1. 保持文档结构一致性，在适当的文件中添加内容
2. 更新README.md中的导航链接（如有新增文档）
3. 使用Markdown格式保持文档样式统一
4. 重要更新同时在更新日志.md中记录
5. 确保文档内容与最新代码保持同步
6. 开发日志记录在 archive/工作日志.md 中，项目状态总结在 项目状态.md 中
