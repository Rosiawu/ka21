# 相关文章自定义属性

本文档说明如何为工具的相关文章设置自定义属性，包括难度级别、内容分类、技能标签和推荐理由。

## 背景

平台中的教程数据来源于各工具的相关文章(`relatedArticles`)。系统会根据文章标题、内容和所属工具自动推断难度级别、内容分类和技能标签。

然而，自动推断并不总是准确的。为了提供更精确的分类，我们引入了自定义属性字段，允许在`tools.json`文件中直接指定这些属性。

## 可用自定义字段

在每个工具的`relatedArticles`数组中，可以为文章添加以下自定义字段：

| 字段名 | 类型 | 描述 | 可选值 |
|-------|------|------|-------|
| `customDifficultyLevel` | 字符串 | 自定义难度级别 | `小白入门`、`萌新进阶`、`高端玩家` |
| `customCategory` | 字符串 | 自定义内容分类 | 任意字符串，建议使用现有分类：`AI大模型`、`AI绘画`、`AI效率`、`商业应用`、`学术研究`、`技术教程`、`飞书多维表格` |
| `customSkillTags` | 字符串数组 | 自定义技能标签 | 任意字符串数组，例如：`["DeepSeek", "提示词工程", "文本生成"]` |
| `customImageUrl` | 字符串 | 自定义图片URL | 图片URL或数据URL |
| `customRecommendReason` | 字符串 | 推荐阅读此文章的理由 | 简短说明，例如：`"最适合新手的AI提示词入门指南，包含实用技巧"` |

## 使用示例

```json
{
  "title": "DeepSeek 应用教程",
  "url": "https://example.com/tutorial",
  "source": "作者名称",
  "publishDate": "2025-02-22",
  "customDifficultyLevel": "小白入门",
  "customCategory": "AI大模型",
  "customSkillTags": ["DeepSeek", "提示词工程", "文本生成"],
  "customImageUrl": "https://example.com/image.jpg",
  "customRecommendReason": "这篇教程通过简单易懂的例子，帮助你快速掌握DeepSeek的核心功能"
}
```

## 优先级规则

1. 如果提供了自定义属性，系统将优先使用自定义属性
2. 如果未提供自定义属性，系统将使用自动推断的属性

## 最佳实践

- 只在自动推断不准确时设置自定义属性
- 尽量使用现有的分类和标签保持一致性
- 难度级别必须是预定义的三个值之一
- 技能标签应该简洁明了，每篇文章建议不超过5个标签

## 验证

添加自定义字段后，可以运行以下命令验证tools.json文件：

```bash
npm run validate-tools
```

如果验证失败，请检查自定义字段是否符合要求。
