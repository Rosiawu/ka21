# 部署指南

本文档详细说明了数字生命卡兹克-KA21工具导航平台的部署流程、环境配置和最佳实践。

## 部署平台

项目使用 Vercel 进行部署，这是一个专为 Next.js 应用优化的云平台，提供了自动化的持续集成和部署流程。

## 部署流程

### 基于 Vercel 的自动部署

1. **准备工作**
   - 确保你有 GitHub/GitLab 账户并且有项目的访问权限
   - 注册或登录 [Vercel](https://vercel.com) 账户
   - 将 Vercel 与你的代码托管平台账户关联

2. **导入项目**
   - 在 Vercel 控制台中点击"Import Project"
   - 选择"Import Git Repository"
   - 找到并选择 ka21-tools 项目仓库
   - 如果你是从 Fork 的仓库部署，请选择你 Fork 的仓库

3. **配置项目**
   - **框架预设**: 选择 Next.js
   - **构建设置**: 保持默认设置 (`npm run build`)
   - **输出目录**: 默认为 `.next`
   - **根目录**: 保持默认值 (`.`)
   - **环境变量**: 如有必要，添加环境变量（见下文）

4. **部署项目**
   - 点击"Deploy"按钮
   - Vercel 将自动构建和部署项目
   - 成功后，你将获得一个可访问的URL

### 持续部署

一旦初始设置完成，Vercel 将自动实现持续部署：

- **自动部署**: 每次推送到主分支（通常是 `main` 或 `master`）时，Vercel 将自动构建和部署最新版本
- **预览部署**: 每个 Pull Request 都会创建一个预览部署，方便在合并前进行测试
- **回滚功能**: 如果部署后发现问题，可以通过 Vercel 控制台快速回滚到之前的版本

## 环境配置

### 环境变量

根据项目需求，您可能需要配置以下环境变量：

- **NEXT_PUBLIC_BASE_URL**: 网站基础URL，用于生成绝对链接
- **NEXT_PUBLIC_API_URL**: API服务的URL（如果有）
- **NODE_ENV**: 运行环境（`development`/`production`）

在 Vercel 中配置环境变量：
1. 进入项目设置
2. 找到"Environment Variables"部分
3. 添加所需的键值对
4. 保存更改并重新部署

### 部署配置

项目的部署配置主要通过以下文件控制：

- **next.config.js**: Next.js 配置，控制路由、图像优化、输出等
- **package.json**: 定义构建和启动脚本
- **vercel.json**: Vercel 特定配置（如果有）

## 自定义域名设置

要将项目部署到自定义域名：

1. 在 Vercel 项目设置中，找到"Domains"部分
2. 点击"Add"并输入您的域名
3. 按照 Vercel 提供的指示，配置您的 DNS 设置
   - 通常需要添加 CNAME 记录或配置 A 记录
4. 验证域名所有权（通常通过 DNS TXT 记录）
5. 等待 DNS 传播完成（可能需要24小时）

## 手动部署选项

除了 Vercel，您还可以选择其他部署方式：

### 静态导出部署

Next.js 支持静态导出，生成可部署到任何静态托管服务的文件：

1. 配置 `next.config.js` 启用输出导出:
```js
module.exports = {
  output: 'export'
}
```

2. 构建项目:
```bash
npm run build
```

3. 生成的静态文件将位于 `out` 目录，可部署到任何静态托管服务:
   - GitHub Pages
   - Netlify
   - AWS S3
   - Firebase Hosting
   - 等

### 传统服务器部署

您也可以在传统服务器上部署 Next.js 应用：

1. 构建项目:
```bash
npm run build
```

2. 将以下文件和目录复制到服务器:
   - `.next/`
   - `public/`
   - `package.json`
   - `next.config.js`

3. 在服务器上安装依赖:
```bash
npm install --production
```

4. 启动服务器:
```bash
npm start
```

5. 配置反向代理（如 Nginx 或 Apache）将流量转发到 Next.js 应用

## 部署最佳实践

### 性能优化

1. **图像优化**
   - 确保使用 Next.js 的图像组件 (`next/image`)
   - 合理设置图像尺寸和质量

2. **代码分割**
   - 利用 Next.js 的自动代码分割功能
   - 使用动态导入 (`dynamic import`) 进一步优化

3. **缓存策略**
   - 配置适当的缓存控制头
   - 利用 Vercel 的边缘缓存

### 安全最佳实践

1. **定期更新依赖**
   - 使用 `npm audit` 检查安全漏洞
   - 及时更新存在安全问题的依赖包

2. **内容安全策略**
   - 配置适当的 CSP 头
   - 限制不必要的外部资源访问

3. **外部链接安全**
   - 为所有外部链接添加 `rel="noopener noreferrer"` 属性

### 监控与维护

1. **性能监控**
   - 使用 Vercel Analytics 或其他工具监控性能
   - 跟踪关键性能指标（LCP、FID、CLS等）

2. **错误跟踪**
   - 实现前端错误跟踪
   - 定期检查错误日志

3. **定期备份**
   - 备份关键数据和配置
   - 定期测试恢复流程

## 故障排除

### 常见部署问题

1. **构建失败**
   - 检查构建日志了解具体错误
   - 确保所有依赖都已正确安装
   - 验证环境变量是否正确配置

2. **页面加载问题**
   - 检查网络请求错误
   - 验证API端点是否可访问
   - 检查浏览器控制台中的JavaScript错误

3. **样式或图像缺失**
   - 验证路径是否正确
   - 确保资源文件已包含在构建中
   - 检查网络请求是否返回404错误
